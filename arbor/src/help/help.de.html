Anleitung!

Mit diesem CODAP Plug-in können Entscheidungsbäume basierend auf Daten ertstellt werden. Das geht folgendermaßen:

  • Erste Schritte: Um einen Baum zu erstellen, zieht man die Variable, die man vorhersagen möchte, in den blauen Bereich der Registerkarte "Baum". Dadurch wird ein Feld mit dem Namen der Variable erstellt. Dies ist der "Wurzelknoten".
  • Um eine Verzweigung zu erstellen, zieht man eine Variable auf einen Knoten.
  • Um eine bestimmte Vorhersage zuzuweisen, klickt man auf die runden "Label". Ein weiterer Klick ändert die Vorhersage.
  • Hiermit gelangt man zu den Einstellungen des ausgewählten Knotens.
  • Statistiken zum aktuellen Baum befinden sich unter dem Baum./li>
  • Löschen: Ein Klick auf den Mülleimer entfernt die nachfolgenden Äste.
  • Um eine neue Zielvariable festzulegen, zieht man die neue Variable zum obersten "Streifen" des Wurzelknotens.
  • Mit einem Klick auf einen Knoten wird dieser und alle zugehörigen Fälle ausgewählt (er wird gelb eingefärbt).

Weitere Details:

Es soll nun festgestellt werden, ob man an einer Krankheit namens Fbola leidet. Dazu gibt es eine Reihe von Tests und Hinweisen, die in Form von Daten vorliegen.

An Fbola erkrankt zu sein, ist die Zielvariable. Es liegen Test-, Untersuchungs- und Krankheitsdaten für verschiedene Personen vor, die sich in der Klinik vorgestellt haben. Der Baum repräsentiert eine Reihe von Fragen, wie zum Beispiel: Haben Sie Husten? Liegt Ihr Fieber über 39 Grad Celsius? Hatten Sie schon einmal die Masern? Das Design des Baums stellt eine Abfolge von Fragen dar, die jeweils von den Antworten auf die vorangegangenen Fragen abhängen.

Nach jeder Frage kann man sich entscheiden, aufzuhören und eine Vorhersage (oder Diagnose) zu treffen. Eine Vorhersage (Diagnose) ist entweder positiv (d. h. man nimmt an, dass Fbola vorliegt) oder negativ. Die positiven und negativen Vorhersagen (Diagnosen) werden am unteren Ende des Baums durch grüne oder braune "Label" dargestellt.

Natürlich liegt man mit einer Vorhersage nicht jedes Mal richtig. Es gibt vier Möglichkeiten:

  • Die Vorhersage lautet "positiv" und die Krankheit liegt tatsächlich vor (Richtig positiv, RP)
  • Die Vorhersage lautet "positiv", aber die Krankheit liegt nicht vor (Falsch positiv, FP)
  • Die Vorhersage lautet "negativ", aber die Krankheit liegt vor (Falsch negativ, FN)
  • Die Vorhersage lautet "negativ" und die Krankheit liegt tatsächlich nicht vor (Richtig negativ, RN)

Die Variablen sollten im Baum so angeordnet werden, dass möglichst viele Fälle richtig vorhergesagt werden.

Variablen (oder Merkmale) und Knoten

Die Namen aller Variablen stehen als Spaltenüberschriften in der Tabelle oder als Zeilen in der Fallkarte.

Um den Baum nach einer Variablen zu verzweigen (was in diesem Beispiel dem Fragen einer Frage entspricht), zieht man die Variable aus der Tabelle auf den Knoten, den man verzweigen möchte.

Jeder Knoten ist entweder eine "Abzweigung" (falls man eine Frage stellt) oder ein "Endknoten" (an welchem man eine Vorhersage oder Diagnose trifft). Der Endknoten enspricht dem vergebenen Label. Label sind entweder "+" oder "–". (Zu Beginn zeigt ein Label nichts an ("?").)

Die Beschriftung eines Endknotens kann man ändern, indem man darauf klickt.

Zielvariable

Der Baum enthält immer eine Zielvariable. Diese soll mit Hilfe der anderen Variablen vorhergesagt werden. Wenn man beispielsweise Daten über Personen hat (Geschlecht, Einkommen, Größe, Schulabschluss) und man das Einkommen von Personen vorhersagen möchte, dann macht man Einkommen zur Zielvariable.

Die Zielvariable erscheint immer an der Spitze des Baums, dem ersten "Wurzelknoten" im oberen "Streifen". Dieser Knoten legt auch fest, welcher Wert der Zielvariable im Baum als "positiv" betrachtet wird.

Um eine Variabel als Zielvariable zu kennzeichnen, zieht man sie zur Spitze auf den "Wurzelknoten" auf den obersten "Streifen".

Positive und negative Label

Immer wenn man dem Baum eine Variable hinzufügt, werden zwei Äste erstellt. Um die Label zu beschriften, wählt man in der Regel "plus" (+) für das eine Label und "Minus" (-) für das andere. Das "+" steht für eine "positive" Wahl, auch wenn es sich um ein negatives oder schlechtes Ergebnis handelt. Beispielsweise möchte man für eine Krankheit keinen positiven Test haben (weil das bedeutet, dass man krank ist), aber das positive Ergebnis führt dazu, dass man vermutet (oder vorhersagt), dass man diese Krankheit tatsächlich hat.

Einstellungen zu Variablen

Alle Verzweigungen im Baum sind binär. Man wählt immer zwischen zwei Möglichkeiten. Die Variable kann jedoch auch mehr als zwei Werte haben. Das Plugin trifft eine Standardauswahl darüber, wie die Werte der Variable den beiden Möglichkeiten zugeordnet werden. Möglicherweise muss man diese Zuordnung von Hand in einem Knoten einstellen und verändern.

Dazu klickt man auf den Button in einem Knoten, um die Einstellungen zu öffnen.

Jede Variable hat einen "linken" und einen "rechten" Wert. Im Allgemeinen möchte man mehr "positive" Fälle (in Bezug auf die Zielvariable) auf der "linken" Seite haben.

  • Bei numerischen Variablen kann man einen Operator (<, >=) und einen Wert festlegen. So wird die "Grenze" (Schwellenwert/Splitwert) definiert, durch die die Fälle dem rechten oder linken Ast zugeordnet werden.
  • Bei kategorialen Variablen kann die Zuweisung der Werte zum rechten oder linken Ast im Konfigurationsmenü geschehen. Dafür klickt man den entsprechenden Namen an, damit er die Seite wechselt. Man kann selbst entscheiden, wie die Äste beschriftet werden sollen, indem man die Textboxen verwendet.

Zwei verschiedene Arten von Entscheidungsbäumen

In der Registerkarte Einstellungen kann man die Art des Entscheidungsbaums auswählen, je nachdem welche Art von Aufgabe man lösen möchte: Klassifikation oder Regression

Klassifikation ist die Standardeinstellung. Das Ergebnis ist eine positive oder negative Vorhersage. In jedem Knoten wird der Anteil der Fälle angezeigt, die in Bezug auf die Zielvariable den "positiven" Wert haben.

Im Fall von Regression wird ein Zahlenwert für die Zielvariable vorhergesagt. Deshalb sollte die Zielvariable numerisch sein. In jedem Knoten wird der Mittelwert für die Zielvariable, bezogen auf alle Fälle in diesem Knoten, dargestellt.

Interaktionen mit CODAP

Wenn man einen Knoten anklickt, werden alle zugehörigen Fälle in CODAP ausgewählt.

Wenn man einen Fall in der Datentabelle in CODAP auswählt, wird der zugehörige Pfad im Baum gelb markiert.

Durch Anklicken von Daten ausgeben werden Informationen über den aktuellen Entscheidungsbaum in einer neuen Datentabelle Daten zum Entscheidungsbaum protokolliert.

Der zugehörige Baum kann wieder hergestellt werden, indem man die passende Zeile in der Tabelle Daten zum Entscheidungsbaum anklickt.
